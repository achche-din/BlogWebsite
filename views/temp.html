What is Inheritance?

Inheritance is the concept of using the functionality offered by another class and building on top of it. This way, multiple new classes can build upon one class which is offering the base functionality.

What actually happens at Inheritance?

Only members of a class that are declared protected or public are inherited by subclasses.

Method bodies aren't copied in the body of a subclass. Instead, when you call 

 Child child1 = new Child(); child1.method1();

It will look through its hierarchy, going up a level every time it can't find an implementation. First, it will check Child which has no implementation. One level above that there's Parent which does, so it will use that one. this is achieved via the super() call. The super keyword is used to refer to the object of the immediate parent class. The very first thing super does is initialize an object of parent class in constructor of the child class as shown in code below. 

 Child() { super(); parent_method(); }

When to apply Inheritance?

  • Both classes are in the same logical domain and The subclass is a proper subtype of the superclass. Example: Sparrow IS-A Bird, Cat IS-A Animal.

  • The superclass’s implementation is necessary or appropriate for the subclass

  • The enhancements made by the subclass are primarily additive.

  • If you find that your subclass is removing things provided by the superclass, question inheriting from that superclass

Advantages of using Inheritance

  • Provide feature of Method overriding which helps in runtime polymorphism.

  • Code reusability: common code for all the derived classes of the parent class can be put in parent class so same code need not be put in multiple child classes hence reducing code redundancy.

  • Inheritance helps to extend features of the base class without affecting base class “We need a widget that is just like the existing Widget class but with a few tweaks and enhancements”

  • Inheritance helps programmer to model IS-A relationship in code which makes code easier to maintain(read, update, debug).

Drawbacks of Inheritance

  • Main disadvantage of using inheritance is that the two classes (base and inherited class) get tightly coupled. This means one cannot be used independently of each other. for example, during maintenance adding new features both base as well as derived classes are required to be changed. If a method signature is changed or a method is deleted in the "superclass", then we will have to re-factor in case of using that method in derived classes.

  • increased time/effort it takes the program to jump through all the levels of overloaded classes. If a given class has ten levels of abstraction above it, then it will essentially take ten jumps to run through a function defined in each of those classes.
