STACK

The stack is a data structure which follows the property of Last In First Out (LIFO)

i.e the last element to be added will be the first element to be removed from the stack. To understand it better let's make a real-life analogy:

we have all visited Library where we find stacks of books like shown in the picture.

so what we have to do in order to remove the book ... yes you are right we need to remove all the books above it. This functionality okay maybe in case of the book it is a limitation but in computer science, it is of utmost importance as you will see later is what makes up the stack that is elements need to removed sequentially from the top in order to reach an element placed below.

                                                  IMPLEMENTATION

Stacks can be implemented using arrays or linked list. Use arrays when we are sure about the maximum size the stack can grow as arrays are static in size. If the requirement is to implement dynamic stack use linked list (dynamic in the sense that the stack size do not have to be fixed it can be variable).

Functions to be defined :

*push(): pushes an element at the top of the stack(at the top of book stack)

*pop(): removes an element from the top of the stack(from the top of book stack).

*peek(): returns the element at the top of the stack without removing it. (getting  the name of the top book on the book stack)

*isEmpty():returns true if the stack is empty(stacks has no books)else returns      false.

you can use java collections library if you do not wish to implement the stack class yourself.

Java Inbuilt Interface : Deque<T> which is implemented by class ArrayDeque

It has function:

*offerLast(): which adds an element at end of the stack(similar to push()).

*pollLast(): which removes element from the top of the stack(similar to pop()).

*isEmpty(): check if stack is empty(similar to isEmpty()).

*peekLast(): return the top of the stack(similar to peek()).

There is a class named stack also in java but it is deprecated so prefer using Deque instead of the class stack.

USES OF STACKS

  • evaluation of arithmetic expression: Arithmetic expression is evaluated by computers by first converting them to postfix or prefix expression which are formed with help of stacks.
  • The undo button in PCs (Ctrl+Z) make use of stacks to go back to the previous state. Backwards and forward in web browsers.
  • Stacks are used for recursion/backtracking which are very useful algorithmic technique. The recursive function parameters and local variables are stored in the stack so that function can continue from where it left off.
  • Braces are checked by compilers using stacks.
  • Many problems are solved using stacks which you will get to know as you solve lots of questions on stacks. One useful stack is a monotone stack which maintains elements in increasing order which is very useful to solve problems.
