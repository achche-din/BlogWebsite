What is Database?

A database is an organized collection of structured information, or data, typically stored electronically in a computer system. Data stored in a database can then be easily accessed, managed, modified, updated, controlled, and organized using a “database management system”.

What is Database?

A database typically requires a comprehensive database software program known as a database management system (DBMS). A DBMS serves as an interface between the database and its end-users or programs, allowing users to retrieve, update, and manage how the information is organized and optimized. Some examples of popular database software or DBMSs include MySQL, Microsoft Access, Microsoft SQL Server, MongoDB.

 

Features of Database Management System

 

  • Data storage, retrieval and update

    • All information in a digital repository is organized into a clear hierarchical structure with records, tables or objects. Every piece of information that you can add to your database will be organized in a kind of catalogue, so it will be easier for you to search and update your records later on.

  • Support for transactions and concurrency

    • Transaction is a collection of operations on the database that are bound together into a single run-time unit. The transaction manager controls and coordinates the execution of transactions within the database. 

    • When multiple users are accessing the database at the same time, and at least one is updating data, there may be the case of interference, which can result in data inconsistencies. Concurrency Manager in DBMS facilitates multiple users to access shared data in parallel without data inconsistencies.

  • Enforcing constraints to ensure data in the database abides by certain rules

    • You can enforce rules by defining integrity constraints more reliably than by adding logic to your application. There are check constraints, cascades, triggers in DBMS which help to keep database consistent.

  • Access support from remote locations

    • Remote database access (RDA) is a protocol standard for database access it includes features for communicating database operations and parameters from the client to the server, in return transporting result data from the server to the client.

 

Challenges to Database Management

 

  • Absorbing significant increases in data volume.

    • The explosion of data coming in from sensors, connected machines, and dozens of other source makes it challenging to organize data efficiently.

  • Ensuring data security

    • Data breaches are happening everywhere these days, and hackers are getting more inventive. It’s more important than ever to ensure that data is secure but also easily accessible to users.

  • Managing and maintaining the database and infrastructure.

    • Database administrators must continually watch the database for problems and perform preventative maintenance, as well as apply software upgrades and patches. 

 

Challenges Question

 

why use Databases instead of a spreadsheet?

Spreadsheets were originally designed for one user. They’re great for a single user or a small number of users who don’t need to do a lot of incredibly complicated data manipulation. Databases, on the other hand, are designed to hold much larger collections of organized information—massive amounts, sometimes. Databases allow multiple users at the same time to quickly and securely access and query the data using highly complex logic and language.

What is OOP?

OOPs stands for 'Object Oriented Programming'. It is a programming paradigm based on the concept of objects. Real-world entities can be modelled using objects which describes its attributes(fields) and behaviour(code which manipulates the attributes). In OOP, computer programs are designed by making them out of objects that interact with one another.

Why OOP?

  • Modularity in Code for easier troubleshooting

    • Something has gone wrong, and you have no idea where to look. With OOP you know exactly where to look. If an object functionality is broken you just have to look at its class and fix that functionality. Encapsulation provides modularity in code.

    • Team members can work on multiple objects simultaneously while minimising the chance that one person might duplicate someone else’s functionality

  • Code reuse using Inheritance

    • In programming, there is a principle of DRY(do not repeat yourself). Code repetition is not good since if you want to change that code you need to change at multiple locations where that code is repeated making the code harder to maintain.

  • Code flexibility

    • Polymorphism provides different functionality from a method based on the situation through method overriding and method overloading. The same method can have different behaviour. suppose we have a method driveCar() sometimes when this method is invoked driveCar() is driving Mercedes and other time it could be BMW based on whether the car is BMW or Mercedes. You can read about it more here.

    • We can hide the implementation details of a method using the concept of Abstraction. Implementators of the class can change implementation and no user of the class is affected as long as method signature remains the same.

  • Object-oriented programming is often the most natural and pragmatic approach, once you get the hang of it. OOP languages allow you to break down your software into bite-sized problems that you then can solve — one object at a time

Job Scheduling

 

Given an array of jobs. Each job has an associated deadline and profit with the job if finished before the deadline. Assume each job takes one unit of time to complete. Only one job can be scheduled at a moment. Your task is to maximise the profit earned by jobs.

Sample Input:
Deadline:   [ 4, 1, 1, 1 ]
Profit:         [ 20, 10, 40, 30 ]

Sample Output:
60

Sample Explanation:
Scheduling jobs {c,a} maximises our profit to 60

Constraints:
0 <= deadline[i] <= 100
0 <= profit[i] <=1 000
1 <= Number of jobs <=100000

Time complexity     O(n log n)

Binary Search Tree

 

Having a sorted array is useful for many tasks because it enables a binary search to be used to efficiently locate elements. The problem with a sorted array is that elements can't be inserted and removed efficiently. The binary search tree is a different way of structuring data so that it can still be binary searched (or a very similar procedure can be used), but it's easier to add and remove elements. Instead of just storing the elements contiguously from least to greatest, the data is maintained in many separate chunks, making adding an element a matter of adding a new chunk of memory and linking it to existing chunks.

 

Properties of BST

  • Binary search tree has a total order defined on it. A total order means a binary comparison operation between elements has been defined, and the elements can be arranged in a unique sequence from smallest to greatest based on this operation. For example, integers or reals with the typical ("natural") order.

  • A binary search tree is a rooted binary tree, whose internal nodes each store a key (and optionally, an associated value), and each has two distinguished sub-trees, commonly denoted left and right. The tree additionally satisfies the binary search property: the key in each node is greater than or equal to any key stored in the left sub-tree, and less than or equal to any key stored in the right sub-tree.

  • An in-order traversal of a binary search tree will always result in a sorted list of node items (numbers, strings or other comparable items).

 

Applications of BST

  • Binary search trees support everything you can get from a sorted array: efficient search, in-order forward/backwards traversal from any given element, predecessor /successor element search, and max /min queries, with the added benefit of efficient inserts and deletes. With a self-balancing binary search tree (BST) like AVL, RBT, all of the above run in logarithmic time.

  • Binary search trees are a fundamental data structure used to construct more abstract data structures such as sets, multisets.

  • Binary serach trees are used in popular data compression technique Huffman coding.
